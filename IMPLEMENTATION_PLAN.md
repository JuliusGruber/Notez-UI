# Notez Angular UI - Implementation Plan

## Overview

Build an Angular single-page application for managing notes with a two-panel layout:
- **Left Panel**: List of all notes (scrollable)
- **Right Panel**: View/Edit note with form

Backend API runs at `http://localhost:8080` during local development.

## Tech Stack

| Technology | Version | Purpose |
|------------|---------|---------|
| Angular | 17/18 (latest) | Frontend framework |
| Angular Material | Latest | UI component library |
| Standalone Components | Yes | Modern Angular architecture |
| SCSS | - | Styling |
| RxJS | - | Reactive programming |

## Key Requirements

- **UI Library**: Angular Material
- **Angular Version**: Latest (17/18) with standalone components
- **Authentication**: None (for now)
- **View/Edit Mode**: Notes can be viewed and edited
- **CRUD Operations**: Create, Read, Update, Delete from the start

## Validation Rules

| Field | Rule | Message |
|-------|------|---------|
| Title | Required | "Title is required" |
| Title | Max 200 characters | "Title cannot exceed 200 characters" |
| Title | Min 1 character (trimmed) | "Title cannot be empty" |
| Content | Required | "Content is required" |
| Content | Max 10,000 characters | "Content cannot exceed 10,000 characters" |

## Error Handling Strategy

| HTTP Status | Action | User Message |
|-------------|--------|--------------|
| 400 Bad Request | Show validation errors | Display server validation messages |
| 404 Not Found | Remove from list, clear selection | "Note not found. It may have been deleted." |
| 500 Server Error | Show error snackbar | "Server error. Please try again later." |
| Network Error | Show error snackbar with retry | "Connection failed. Check your network." |

### Retry Logic
- Automatic retry: None (user-initiated only)
- Retry button shown on network errors
- Exponential backoff not needed for simple CRUD app

## Responsive Design

### Breakpoints

| Breakpoint | Width | Layout |
|------------|-------|--------|
| Desktop | > 960px | Two-panel side-by-side (35% / 65%) |
| Tablet | 600px - 960px | Two-panel (40% / 60%) |
| Mobile | < 600px | Single panel with navigation |

### Mobile Behavior
- Notes list shown by default
- Selecting a note navigates to detail view (full screen)
- Back button returns to list
- FAB (Floating Action Button) for "Add Note" on mobile

## Phase 4: State Management & Data Flow

### State Management Approach: Angular Signals

Using Angular 17+ Signals for reactive state management (simpler than NgRx for this use case):

```typescript
import { signal, computed } from '@angular/core';

@Injectable({ providedIn: 'root' })
export class NotesStateService {
  // Core state using signals
  private _notes = signal<Note[]>([]);
  private _selectedNoteId = signal<number | null>(null);
  private _editMode = signal<'view' | 'edit' | 'create'>('view');
  private _isLoading = signal(false);
  private _hasUnsavedChanges = signal(false);

  // Public read-only signals
  readonly notes = this._notes.asReadonly();
  readonly selectedNoteId = this._selectedNoteId.asReadonly();
  readonly editMode = this._editMode.asReadonly();
  readonly isLoading = this._isLoading.asReadonly();
  readonly hasUnsavedChanges = this._hasUnsavedChanges.asReadonly();

  // Computed signal for selected note
  readonly selectedNote = computed(() => {
    const id = this._selectedNoteId();
    return this._notes().find(n => n.id === id) ?? null;
  });

  // State mutations
  setNotes(notes: Note[]) { this._notes.set(notes); }
  selectNote(id: number | null) { this._selectedNoteId.set(id); }
  setEditMode(mode: 'view' | 'edit' | 'create') { this._editMode.set(mode); }
  setLoading(loading: boolean) { this._isLoading.set(loading); }
  setUnsavedChanges(dirty: boolean) { this._hasUnsavedChanges.set(dirty); }
}
```

### Unsaved Changes Warning

Warn user when navigating away with unsaved edits:

```typescript
import { CanDeactivate } from '@angular/router';

export const unsavedChangesGuard: CanDeactivate<MainLayoutComponent> = (component) => {
  if (component.notesState.hasUnsavedChanges()) {
    return confirm('You have unsaved changes. Are you sure you want to leave?');

  return true;
```

### Handling Note Selection with Unsaved Changes

```typescript
onNoteSelected(note: Note): void {
  if (this.notesState.hasUnsavedChanges()) {
    const dialogRef = this.dialog.open(ConfirmDialogComponent, {
      data: {
        title: 'Unsaved Changes',
        message: 'You have unsaved changes. Discard them?',
        confirmText: 'Discard',
        cancelText: 'Keep Editing'
      }
    });

    dialogRef.afterClosed().subscribe(confirmed => {
      if (confirmed) {
        this.notesState.setUnsavedChanges(false);
        this.selectNoteAndNavigate(note);
      }
    });
  } else {
    this.selectNoteAndNavigate(note);
  }
}
```

### Sort Options

Default sort: **Updated date (newest first)**

```typescript
// In NotesStateService
private _sortBy = signal<'updatedAt' | 'createdAt' | 'title'>('updatedAt');
private _sortOrder = signal<'asc' | 'desc'>('desc');

readonly sortedNotes = computed(() => {
  const notes = [...this._notes()];
  const sortBy = this._sortBy();
  const order = this._sortOrder();

  return notes.sort((a, b) => {
    let comparison = 0;
    if (sortBy === 'title') {
      comparison = a.title.localeCompare(b.title);
    } else {
      comparison = new Date(a[sortBy] ?? 0).getTime() - new Date(b[sortBy] ?? 0).getTime();
    }
    return order === 'desc' ? -comparison : comparison;
  });
});
```

### Filter with Debounce

```typescript
// In NotesListComponent
searchControl = new FormControl('');

filteredNotes = computed(() => {
  const search = this.searchTerm().toLowerCase().trim();
  const notes = this.notesState.sortedNotes();

  if (!search) return notes;

  return notes.filter(note =>
    note.title.toLowerCase().includes(search) ||
    note.content.toLowerCase().includes(search)
  );
});

constructor() {
  // Debounce search input
  this.searchControl.valueChanges.pipe(
    debounceTime(300),
    distinctUntilChanged()
  ).subscribe(value => this.searchTerm.set(value ?? ''));
}
```
